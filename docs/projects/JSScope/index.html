JavaScript MicroScope

This is a scoping visualizer that allows you to see how JS scoping rules function.
On the left you can create JS programs in a limited and fragile subset of javascript.
The rules for this language are as follows:

  • The only valid lines of code are:
    • variable decelerations of the form "var thingy = 4;"
    • function definitions of the form "function a(){"", notably this only works if the closing brace falls of a line after the opening brace.
    • the closing of a function definition which is always formed like "}"
    • variable modifications of the form "thingy = '7';"
    • function calls of the form "a();"
    • console.log's of the form "console.log('hi')"" or "console.log(thingy)"
  • all lines of code except closing brackets must end with a semi colon
I believe that if you follow these rules then the JS program will be run correctly.
And at each stage in the process the current scope frames will be shown on the right.
